# FrextCloud – Phase 1 (Core Hosting Hub)

## Goal

Build and launch a **minimal viable hosting platform** similar to Hostinger/Vercel where users can:

* Sign up / log in
* Upload and deploy **static websites** (HTML, CSS, JS, React build, Next.js static export)
* Receive a **free subdomain** like:

```
myapp.frextcloud.com
```

* Manage their projects inside a **dashboard**

This phase focuses on **simplicity and fast deployment**.
No Git integration, no custom domains yet.

**Target Timeline:** 2–4 weeks

---

# Platform Overview

FrextCloud Phase 1 will act as a **basic static site hosting platform**.

Users will be able to:

* Create projects
* Upload ZIP files containing website builds
* Automatically deploy them
* Access them via subdomain

Example live URL:

```
https://myapp.frextcloud.com
```

# Deployment System

Deployment process:

```
ZIP upload
↓
Extract files
↓
Build Docker container
↓
Run container
↓
Assign subdomain
↓
Configure Nginx
↓
Enable SSL
```

Example container mapping:

```
myapp → container port 8080
```

---

# Docker Setup

Each project runs inside its own container.

Example Dockerfile:

```
FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80
```

---

# Nginx Reverse Proxy

Handles subdomain routing.

Example configuration:

```
server {
    server_name myapp.frextcloud.com;

    location / {
        proxy_pass http://localhost:8080;
    }
}
```

Wildcard routing will allow unlimited subdomains.

Purchase domain:

```
frextcloud.com
```

Use DNS provider:

DNS records:

```
A record
frextcloud.com → server IP
```

Wildcard record:

```
*.frextcloud.com → server IP
```

This allows unlimited subdomains.
